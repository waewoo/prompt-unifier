# Global Conventions

This Skill provides Claude Code with specific guidance on how to adhere to coding standards as they relate to how it should handle global conventions.

## When to use this skill

- When creating new files or directories anywhere in the project
- When structuring modules, packages, or code organization
- When writing Git commit messages following conventional commit standards
- When organizing code architecture and following established patterns
- When ensuring consistency with existing codebase conventions and practices
- When naming files, folders, and directories across the project
- When deciding on project structure and directory hierarchies
- When following architectural patterns (MVC, layered, modular, etc.)
- When adhering to team workflows and development practices
- When maintaining consistency with existing code patterns and styles
- When creating configuration files or environment variables
- When organizing test files and test directory structures
- When setting up new features that should follow existing patterns
