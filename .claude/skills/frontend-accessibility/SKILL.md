# Frontend Accessibility

This Skill provides Claude Code with specific guidance on how to adhere to coding standards as they relate to how it should handle frontend accessibility.

## When to use this skill

- When building interactive UI components (buttons, forms, modals, dropdowns)
- When adding ARIA labels, roles, and properties to enhance accessibility
- When implementing keyboard navigation and focus management
- When ensuring color contrast meets WCAG AA or AAA standards
- When creating forms with proper labels, fieldsets, and error messages
- When testing components with screen readers (NVDA, JAWS, VoiceOver)
- When working with interactive elements like buttons, links, or custom controls
- When implementing custom components that need accessibility support
- When adding descriptive alt text to images and media
- When managing focus states and focus trapping (in modals, dialogs)
- When ensuring accessible error messages, notifications, and validation feedback
- When implementing skip links or landmark regions for navigation
- When creating accessible data tables with proper headers and structure
